# 매입단가 시트 기능 안내 - 템플릿 채우기 방식

## 핵심 변경

이 버전은 프로그램에서 매입단가 시트를 새로 만들거나 디자인을 다시 입히지 않습니다.
사용자가 제공한 `채소팜_매입단가_시트_템플릿.xlsx`를 먼저 구글시트로 가져온 뒤, 프로그램은 그 템플릿 안에 값만 채워 넣습니다.

## 환경변수

기존 재고파악 시트는 기존 `SPREADSHEET_ID`를 그대로 사용합니다.
매입단가 템플릿 구글시트는 아래 환경변수를 추가해서 분리합니다.

```text
PURCHASE_SPREADSHEET_ID=매입단가 템플릿 구글시트 ID
```

매입단가 템플릿 구글시트도 기존 재고파악 시트와 마찬가지로 서비스 계정 이메일에 편집자 권한으로 공유해야 합니다.

## 사용 순서

1. 엑셀 템플릿 파일을 구글 드라이브에 업로드합니다.
2. 업로드한 엑셀 파일을 Google 스프레드시트로 엽니다.
3. 그 구글시트의 URL에서 스프레드시트 ID를 복사합니다.
4. Render 환경변수 `PURCHASE_SPREADSHEET_ID`에 해당 ID를 넣습니다.
5. 서비스 계정 이메일을 해당 구글시트에 편집자로 공유합니다.
6. Render 재배포 후 프로그램을 사용합니다.

## 프로그램 화면 버튼

매입단가 시트 관리 영역 버튼 순서:

```text
[미등록상품 재처리] [상품명 일괄변경 실행]
```

매입단가 템플릿 확인 버튼은 화면에서 제거했습니다.
거래명세서 링크/HTML 실행 시 연결된 템플릿에 값만 입력하며, 디자인과 기존 데이터는 초기화하지 않습니다.
필수 탭은 아래와 같습니다.

- 매입기록
- 품목환산표
- 원물단가표
- 미등록상품
- 상품명변경
- 전표관리
- 전표상세관리
- 설정

## 처리 흐름

거래명세서 링크 또는 HTML 파일을 실행하면:

1. 기존 재고파악 시트 기능은 그대로 실행
2. 매입단가 템플릿의 품목환산표 기준으로 원본품명 → 변환품명 변환
3. 매입기록에 매입일/상품명/매입량/매입총액/단가/월 입력
4. 품목환산표에 없거나 환산수량이 없는 상품은 미등록상품에 기록
5. 전표관리/전표상세관리로 중복 업로드 방지

## 템플릿 보호 방식

- 시트를 새로 만들지 않습니다.
- 컬럼 너비, 색상, 병합 등 디자인을 다시 덮어쓰지 않습니다.
- 기존 매입기록 데이터는 삭제하지 않습니다.
- `설정` 시트는 드롭다운 목록용 값만 갱신합니다.
- `원물단가표`는 조회 수식이 없거나 오류일 때만 수식을 보완합니다.
- `전표상세관리`, `설정` 시트는 관리용이라 숨김 처리될 수 있습니다.

## 주의사항

- 품목환산표에서 단위 칸을 비우면 원본품명만으로 매칭합니다.
- 단위를 적으면 거래명세서 단위와 일치할 때만 매칭합니다.
- 1개당 환산수량은 반드시 입력해야 합니다.

## 원물단가표 조회 조건

- `날짜 선택`은 기본값이 `선택안함`입니다.
- `선택안함` 상태에서는 `월 선택`과 `상품명 선택` 조건으로 조회합니다.
- 날짜를 직접 선택하거나 입력하면 날짜 조건이 우선 적용되며, `월 선택` 값은 무시됩니다.
  - 예: 월 선택이 `6월`, 날짜 선택이 `2026-07-02`이면 `2026-07-02` 기준으로 조회합니다.
- 상품명 선택은 날짜 선택 여부와 관계없이 함께 적용됩니다.
