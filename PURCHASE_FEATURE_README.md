# 매입단가 시트 기능 추가 안내

## 환경변수

기존 재고파악 시트는 기존 `SPREADSHEET_ID`를 그대로 사용합니다.
새 매입단가 시트는 아래 환경변수를 추가해서 분리합니다.

```text
PURCHASE_SPREADSHEET_ID=매입단가 구글시트 ID
```

매입단가 구글시트는 기존 재고파악 시트와 마찬가지로 서비스 계정 이메일에 편집자 권한으로 공유해야 합니다.

## 프로그램 화면 버튼

매입단가 시트 관리 영역 버튼 순서:

```text
[미등록상품 재처리] [상품명 일괄변경 실행] [매입단가 시트 만들기]
```

`매입단가 시트 만들기` 버튼을 누르면 백지 구글시트에 아래 탭을 자동 생성하고, 엑셀 템플릿 스타일에 맞춰 색상/너비/병합/드롭다운/서식을 적용합니다.

- 매입기록
- 품목환산표
- 원물단가표
- 미등록상품
- 상품명변경
- 전표관리
- 전표상세관리
- 설정

## 디자인 반영

`매입단가 시트 만들기`는 기존 데이터를 초기화하지 않습니다. 다만 `원물단가표`는 조회용 화면이라 검색 영역과 수식 레이아웃을 다시 정리합니다.

적용되는 주요 디자인:

- 진한 청록색 헤더, 흰색 굵은 글씨
- 품목환산표/매입기록 등 주요 시트 컬럼 너비 지정
- 원물단가표 상단 검색 영역 병합 및 초록 라벨 적용
- 날짜/수량/금액/단가 표시 형식 적용
- 원물단가표 월/상품명/날짜 선택 드롭다운 적용
- 전표상세관리, 설정 시트 숨김 처리

## 처리 흐름

거래명세서 링크 또는 HTML 파일을 실행하면:

1. 기존 재고파악 시트 기능은 그대로 실행
2. 매입단가 시트의 품목환산표 기준으로 원본품명 → 변환품명 변환
3. 매입기록에 매입일/상품명/매입량/매입총액/단가/월 입력
4. 품목환산표에 없거나 환산수량이 없는 상품은 미등록상품에 기록
5. 전표관리/전표상세관리로 중복 업로드 방지

## 주의사항

- 품목환산표에서 단위 칸을 비우면 원본품명만으로 매칭합니다.
- 단위를 적으면 거래명세서 단위와 일치할 때만 매칭합니다.
- 1개당 환산수량은 반드시 입력해야 합니다.
