# 매입단가 시트 기능 추가 안내

## 추가된 환경변수

Render Web Service에 아래 환경변수를 추가하세요.

```text
PURCHASE_SPREADSHEET_ID=매입단가 시트의 구글시트 ID
```

기존 `SPREADSHEET_ID`는 그대로 **재고파악 시트**로 사용합니다.
새 `PURCHASE_SPREADSHEET_ID`는 별도 **매입단가 시트**로 사용합니다.

## 매입단가 시트 구성

프로그램 화면에서 **[매입단가 시트 기본 구성 확인]** 버튼을 누르면 아래 시트가 자동 생성/확인됩니다.

- 매입기록
- 품목환산표
- 원물단가표
- 미등록상품
- 상품명변경
- 전표관리
- 전표상세관리
- 드롭다운목록

`전표상세관리`, `드롭다운목록`은 숨김 처리됩니다.

## 품목환산표 컬럼

| 원본품명 | 변환품명 | 단위 | 1개당 환산수량 | 사용여부 |
|---|---|---|---:|---|
| 바실 | 바질 | KG | 1 | 사용 |
| 양상추(국산) | 양상추 | BOX | 12 | 사용 |
| 방울토마토 | 방울토마토 | BOX | 9 | 사용 |

- 거래명세서의 원본품명을 기준으로 찾습니다.
- 변환품명이 매입기록의 상품명으로 들어갑니다.
- 원본품명은 중복되면 안 됩니다.
- 같은 변환품명에 원본품명 여러 개를 연결하는 것은 가능합니다.
- 사용여부를 `미사용`으로 두면 매입기록에는 입력하지 않고 제외 처리합니다.

## 매입기록 컬럼

| 매입일 | 상품명 | 매입량 | 매입총액 | 단가 | 월 |
|---|---|---:|---:|---:|---|

- 매입량은 수기로 수정해도 됩니다.
- 단가는 `매입총액 ÷ 매입량` 수식으로 계산됩니다.
- 매입일은 연도 포함, 월은 `7월` 형식입니다.

## 미등록상품 처리

거래명세서에 있는 원본품명이 품목환산표에 없거나 환산수량이 없으면 `미등록상품` 시트에 기록됩니다.

품목환산표에 등록한 뒤 프로그램 화면에서 **[미등록상품 재처리]** 버튼을 누르면 처리대기 항목만 다시 확인해서 매입기록에 추가합니다.

## 상품명 일괄변경

`상품명변경` 시트에 아래처럼 입력하고 프로그램 화면에서 **[상품명 일괄변경 실행]** 버튼을 누릅니다.

| 기존상품명 | 변경상품명 |
|---|---|
| 바질 | 바질잎 |

- 매입기록의 상품명만 변경됩니다.
- 매입일, 매입량, 매입총액, 단가, 월은 변경되지 않습니다.
- 성공한 변경 행은 자동 삭제됩니다.
- 실패한 행은 `오류내용`에 사유가 남습니다.

## 거래명세서 입력

기존 기능처럼 둘 다 지원합니다.

1. 마켓봄 거래명세서 링크
2. 거래명세서 HTML 파일 업로드

둘 중 무엇을 사용해도 재고파악 시트 반영과 매입단가 시트 입력이 함께 실행됩니다.
